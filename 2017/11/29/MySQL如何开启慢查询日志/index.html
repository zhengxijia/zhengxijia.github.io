简介

MySQL的慢查询日志是MySQL提供的一种日志记录，它用来记录在MySQL中响应时间超过阈值的语句，具体指运行时间超过long_query_time值（默认为10s）的SQL，则会被记录到慢查询日志中，结合explain进行全面分析。

基本操作

说明

默认情况下，MySQL数据库没有开启慢查询日志，需要我们手动来设置这个参数。如果不是调优需要的话，一般不启动该参数，因为开启慢查询日志会或多或少带来一定的性能影响。慢查询日志支持将日志记录写入文件。

开启

默认：SHOW VARIABLES LIKE ‘%slow_query_log%’;

开启：set global slow_query_log=1;

再次查看：SHOW VARIABLES LIKE ‘%slow_query_log%’;

永久生效：修改配置文件my.cnf

重启即生效

日志分析工具mysqldumpslow

简介

在生产环境中，如果要手工分析日志，查找、分析SQL，显然是个体力活，MySQL提供了日志分析工具mysqldumpslow。

帮助信息

  • mysqldumpslow –help
  • s：是表示按照何种方式排序
  • c：访问次数
  • I：锁定时间
  • r：返回记录
  • t：查询时间
  • al：平均锁定时间
  • ar：平均返回记录数
  • at：平均查询时间
  • t：即为返回前面多少条的数据
  • g：后边搭配一个正则匹配模式，大小写不敏感
